# Chef AI – Ingredient to Recipe Generator 👨‍🍳

A Streamlit app that turns a list of ingredients into a fun, personalised recipe card — powered by Google's Gemini API.

---

## ✨ Features

- **Flexible Input:** Type ingredients directly or upload a `.txt` or `.json` file.
- **AI-Powered Generation:** Recipes are generated by the Gemini API (`gemini-2.5-flash`), producing realistic quantities, techniques, and cook times tailored to your exact ingredients.
- **Rich UI Cards:** Styled recipe card layouts with titles, steps, servings, and tags.
- **Secure Handling:** Built-in input sanitisation and data validation before anything reaches the AI model.
- **Personality Layer:** Fun titles, context-aware step emojis, and cheerful notes added on top of the raw AI output.
- **Vibrant UI:** Interactive floating food emoji animations (🍅🥕🍝).

## 🚀 How to Install

Clone the repository and install the required dependencies:

```bash
pip install -r requirements.txt
```

## 🔑 API Key Setup

This app calls the Gemini API, so you'll need your own API key before running it.

1. Get a free API key from [Google AI Studio](https://aistudio.google.com/apikey).
2. Create a file named `.env` in the `chef-ai-recipe-generator/` folder (next to `app.py`):

```
GEMINI_API_KEY=your_key_here
```

3. **Never commit this file.** Make sure `.env` is listed in `.gitignore` before pushing to GitHub.

## ▶️ How to Run

Launch the Streamlit server from your terminal:

```bash
streamlit run app.py
```

Once running, open your browser and navigate to http://localhost:8501

## 🥘 Sample Input

You can type the following plain text into the app:

```
tomato, onion, pasta, garlic, olive oil
```

Alternatively, upload the provided `sample_ingredients.txt` file using the built-in file uploader.

## 🏗️ Architecture

```
Streamlit UI (app.py)
       │
       ▼
  mcp_server.py  ← Orchestrates the pipeline
       │
       ├─► security.py          – Sanitise & validate
       ├─► ingredient_parser.py – Parse & normalise
       ├─► recipe_generator.py  – Gemini API recipe generation
       └─► narrator.py          – Add fun personality & emojis
```

Each module is written as a clean Python file with short, focused functions — no classes, no database, and a single external dependency (the Gemini API) isolated entirely to `recipe_generator.py`.

## 📝 Notes

- **Limits:** Maximum of 30 ingredients supported per single request.
- **File Formats:** Strictly accepts `.txt` and `.json` files.
- **JSON Structure:** JSON files must be structured either as a flat list:

  ```json
  ["tomato", "onion"]
  ```

  Or as an object containing an `"ingredients"` key:

  ```json
  { "ingredients": ["tomato", "onion"] }
  ```

## ⚙️ Core Technical Constraints

- **Payload Threshold:** Caps requests at 30 individual ingredients to keep generation fast and the UI responsive.
- **Extension Constraints:** Rejects any uploaded file outside explicit `.txt` or `.json` extensions.
- **API Key Required:** The app will not run without a valid `GEMINI_API_KEY` in `.env` — there is currently no offline fallback mode.
